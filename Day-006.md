✅ Problems Solved

1. Two Sum

Platform: LeetCode

Approach:

- Stored previously seen numbers in a Hash Map.
- Checked if the required complement already existed.

Time Complexity: O(n)

Space Complexity: O(n)

---

2. Group Anagrams

Platform: LeetCode

Approach:

- Used the sorted version of each string as a key.
- Grouped all strings with the same sorted representation.

Time Complexity: O(n × k log k)

Space Complexity: O(n × k)

---

3. Top K Frequent Elements

Platform: LeetCode

Approach:

- Counted element frequencies using a Hash Map.
- Retrieved the top K elements using Bucket Sort or a Heap.

Time Complexity: O(n)

Space Complexity: O(n)

---

4. Longest Consecutive Sequence

Platform: LeetCode

Approach:

- Stored all numbers in a Hash Set.
- Started counting only from numbers that were the beginning of a sequence.

Time Complexity: O(n)

Space Complexity: O(n)

---

🧠 Key Learnings

- Hash Maps reduce many brute-force O(n²) solutions to O(n).
- Frequency counting is one of the most common interview techniques.
- Hash Sets are useful for fast membership checks.
- Combining Hashing with Prefix Sum unlocks many advanced interview problems.
- Choosing the right data structure is often the key to an optimal solution.
