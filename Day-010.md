## ✅ Problems Solved

### 1. Implement Queue using Stacks

**Platform:** LeetCode

**Approach:**

* Used two stacks (`input` and `output`) to simulate queue operations.
* Transferred elements only when necessary to achieve amortized O(1) operations.

**Time Complexity:** O(1) *(Amortized)*

**Space Complexity:** O(n)

---

### 2. Number of Recent Calls

**Platform:** LeetCode

**Approach:**

* Used a queue to maintain only the requests within the last 3000 milliseconds.
* Removed outdated requests from the front.

**Time Complexity:** O(1) *(Amortized)*

**Space Complexity:** O(n)

---

### 3. Dota2 Senate

**Platform:** LeetCode

**Approach:**

* Maintained two queues to track the positions of each party.
* Simulated the voting process until one queue became empty.

**Time Complexity:** O(n)

**Space Complexity:** O(n)

---

### 4. Sliding Window Maximum

**Platform:** LeetCode

**Approach:**

* Used a Monotonic Deque to keep track of useful elements in decreasing order.
* The front of the deque always represented the maximum element of the current window.

**Time Complexity:** O(n)

**Space Complexity:** O(k)

---

## 🧠 Key Learnings

* Queues follow the First In, First Out (FIFO) principle.
* Deques allow insertion and deletion from both ends in O(1).
* Monotonic Queues optimize sliding window maximum/minimum problems.
* Choosing the right queue implementation improves efficiency significantly.
